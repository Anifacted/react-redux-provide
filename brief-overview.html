Brief Overview

Brief Overview

We will be using the following throughout this library's documentation:

  • ECMAScript 2015 - Modern JavaScript. The programming language used in every example.
  • JSX - A JavaScript syntax extension resembling XML, typically used with React.
  • React - A JavaScript library for building user interfaces (UI).
  • Redux - A JavaScript library for managing application state.

We will cover the basics of each as we go along, so if any of this is new to you, you may still understand everything.

This library revolves around three principles:

  1. The UI should be as declarative as possible, a simple composition of components. Ideally, each component is a pure, stateless function. A pure, stateless function component will always render the same output when given the same input. This enforces predictable, well-defined components.

  2. The UI components should always specify the types of properties they use, as propTypes. Components use these properties to render and/or update the application state. A component's properties come from their parent component or a provider instance. Provider instances exist within the context of the application.

  3. Use any number of Redux stores to manage the state of any number of objects. An object can represent a user, a document, anything. A provider is a composition of functions and properties designed to manage these objects. These functions and properties are specific to Redux - e.g., actions, reducers, etc. You can have any number of instances of the same provider. This helps enforce separation of concerns and reusability.

    Note: Providers instantiate as React components need them. Each provider instance has its own Redux store. This store is automatically created based on the provider's properties. You will almost never have to directly access the store.
