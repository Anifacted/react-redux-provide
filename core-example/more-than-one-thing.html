More Than One "Thing"

More Than One "Thing"

Suppose you want to manage more than one thing. All you do is add a key function to the provider definition. This key function will receive each matched component instance. It might look something like this:

const key = ({ props }) => `things/${props.thingId || ''}`;

We'll need to include this key function with the exported provider object:

// src/providers/thing.js

const SET = 'SET';

const key = ({ props }) => `things/${props.thingId || ''}`;

const actions = {
  setThing: (thing) => ({ type: SET, thing })
};

const reducers = {
  thing: (state = 'nothing', { type, thing }) => type === SET ? thing : state
};

export default { key, actions, reducers };

This allows us to have a different provider instance assigned to a component depending on the thingId property passed to it.

Now let's update our components' propTypes to expect the thingId prop. We'll also render the thingId for the sake of obviousness:

// src/components/SomeThing.js

import React, { PropTypes } from 'react';

const SomeThing = ({ thingId, thing }) => (
  <div>
    Thing {thingId} is currently: {thing}
  </div>
);

SomeThing.propTypes = {
  thingId: PropTypes.string.isRequired,
  thing: PropTypes.any.isRequired
};

export default SomeThing;

// src/components/SomeThingSetter.js

import React, { PropTypes } from 'react';

const SomeThingSetter = ({ thingId, setThing }) => (
  <button onClick={() => {
    const thing = window.prompt(`What should thing ${thingId} be?`);
    setThing(thing);
  }}>
    Click me to set thing {thingId}.
  </button>
);

SomeThingSetter.propTypes = {
  thingId: PropTypes.string.isRequired,
  setThing: PropTypes.func.isRequired
};

export default SomeThingSetter;

And then to put it all together as actual rendered components:

// src/components/Things.js

import React, { PropTypes } from 'react';
import SomeThing from './SomeThing';
import SomeThingSetter from './SomeThingSetter';

const Things = () => (
  <div>
    <SomeThing thingId="A" />
    <SomeThingSetter thingId="A" />

    <SomeThing thingId="B" />
    <SomeThingSetter thingId="B" />

    <SomeThing thingId="C" />
    <SomeThingSetter thingId="C" />
  </div>
);

export default Things;

This would ultimately produce the following HTML:

<div>
  <div>
    Thing A is currently: nothing
  </div>
  <button>
    Click me to set thing A.
  </button>

  <div>
    Thing B is currently: nothing
  </div>
  <button>
    Click me to set thing B.
  </button>

  <div>
    Thing C is currently: nothing
  </div>
  <button>
    Click me to set thing C.
  </button>
</div>

Clicking each button would of course allow us to update the current state of each thing.

Note that we can reuse these same components and/or propTypes (actions/reducers) any number of times. The application will use the same provider instance for same values returned by the key function.
