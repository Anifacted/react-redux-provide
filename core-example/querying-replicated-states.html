Querying Replicated States

Querying Replicated States

You may not care about this topic until later on, but it's good to know about sooner rather than later.

Most data sources allow you to query for stored data. In our case, this data represents the states of any number of Redux stores, created by our providers.

Suppose we want to find each thing with a current state of "apple" within this data source. To enable this within our React components, we first need to add a handleQuery function to our replicator. This function works like getInitialState, but instead of a setState function, it receives a setResult function. It also receives a query object and an options object. The store passed to it is typically a "base" store for any matching reducers but its key would be without, for example, a thingId.

const toParams = obj => Object.keys(obj)
  .map(key => `${encodeURIComponent(key)}=${encodeURIComponent(obj[key])}`)
  .join('&');

const handleQuery = ({ store, query, setResult, setError }) => (
  fetch(`/api/${store.key}?${toParams(query)}`)
    .then(response => response.json())
    .then(setResult)
    .catch(setError)
);

We will need to export the handleQuery function along with getInitialState and onStateChange:

// src/replicators/backend.js

const getInitialState = ({ store, setState, setError }) => (
  fetch(`/api/${store.key}`)
    .then(response => response.json())
    .then(setState)
    .catch(setError)
);

const onStateChange = ({ store, nextState, create, setState, setError }) => (
  fetch(`/api/${store.key}`, {
    method: create ? 'POST' : 'PATCH',
    body: JSON.stringify(nextState),
    headers: {
      'Accept': 'application/json',
      'Content-Type': 'application/json'
    }
  })
    .then(response => response.json())
    .then(create || setState)
    .catch(setError)
);

const toParams = obj => Object.keys(obj)
  .map(key => `${encodeURIComponent(key)}=${encodeURIComponent(obj[key])}`)
  .join('&');

const handleQuery = ({ store, query, setResult, setError }) => (
  fetch(`/api/${store.key}?${toParams(query)}`)
    .then(response => response.json())
    .then(setResult)
    .catch(setError)
);

export default { getInitialState, onStateChange, handleQuery };

Now querying for states from within our React components is just a matter of specifying a query prop type and a result prop type:

// src/components/Apples.js

import React, { PropTypes } from 'react';
import SomeThing from './SomeThing';

const Apples = ({ result }) => {
  if (result === null) {
    return (
      <div>
        Finding apples...
      </div>
    );
  } else if (!result || !result.length) {
    return (
      <div>
        No apples found!
      </div>
    );
  } else {
    return (
      <div>
        {result.map(thingState => (
          <SomeThing
            key={`SomeThing:${thingState.thingId}`}
            { ...thingState }
          />
        ))}
      </div>
    );
  }
};

Apples.propTypes = {
  query: PropTypes.any,
  result: PropTypes.any
};

Apples.defaultProps = {
  query: { thing: 'apple' }
};

export default Apples;

The query prop may also be a function. Suppose we want a generic ThingFinder component. Everything remains the same except for the query prop, which becomes a function of the component itself.

// src/components/ThingFinder.js

import React, { PropTypes } from 'react';
import SomeThing from './SomeThing';

const ThingFinder = ({ thing, result }) => {
  if (result === null) {
    return (
      <div>
        Finding {thing}...
      </div>
    );
  } else if (!result || !result.length) {
    return (
      <div>
        No {thing} found!
      </div>
    );
  } else {
    return (
      <div>
        {result.map(thingState => (
          <SomeThing
            key={`SomeThing:${thingState.thingId}`}
            { ...thingState }
          />
        ))}
      </div>
    );
  }
};

ThingFinder.propTypes = {
  thing: PropTypes.string.isRequired,
  query: PropTypes.any,
  result: PropTypes.any
};

ThingFinder.defaultProps = {
  query: ({ props }) => ({ thing: props.thing })
};

export default ThingFinder;

We could then use this generic ThingFinder component to find any thing:

// src/components/Apples.js

import React, { PropTypes } from 'react';
import ThingFinder from './ThingFinder';

const Apples = () => (
  <ThingFinder thing="apple" />
);

export default Apples;
