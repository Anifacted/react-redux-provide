Tying Everything Together

Tying Everything Together

As you may have noticed, the components don't depend on (import) the providers. The providers don't depend on the components or the replicators. And the replicators don't depend on the components or the providers. Each piece of the application is as decoupled as possible.

With each piece so separate from one another, how do we put it all together?

When mounting the app, we'll import the replicator(s), the providers, and the top level component. We'll use react-redux-provide's pushReplicator utility function. We'll also use its install method, which automatically enables providers for all components. Note that the install method may not be necessary in the future when React's context has matured.

import 'react-redux-provide/lib/install';
import React from 'react';
import { render } from 'react-dom';
import { pushReplicator } from 'react-redux-provide';
import backend from './replicators/backend';
import * as providers from './providers/index';
import App from './components/App';

Notice that we're importing the providers from an index module. It's good practice to use an index module which exports each provider within the app. We have just one provider in this particular example, but most applications will use more than a handful.

// src/providers/index.js

export thing from './thing';

Last but not least, all we need to do now is push our backend replicator onto our providers and pass our providers as a property to our top-level component.

// src/renderApp.js

import 'react-redux-provide/lib/install';
import React from 'react';
import { render } from 'react-dom';
import { pushReplicator } from 'react-redux-provide';
import backend from './replicators/backend';
import * as providers from './providers/index';
import App from './components/App';

pushReplicator(providers, backend);

function renderApp(props, element = document.getElementById('root')) {
  return render(<App { ...props } />, element);
}

renderApp({ providers });

export default renderApp;

That's it! The concepts covered throughout this example are what you'll use the most.

If you're ready for more advanced use-cases, checkout the Extended Documentation.
